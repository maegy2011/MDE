# ⚖️ الدليل الطبي الذكي (Medical Directory App)

تطبيق ويب تقدمي (PWA) متطور ومستقل، تم تطويره ليكون **"وثيقة داخلية مؤمنة"** (مخصص للمؤسسات الكبرى). يهدف التطبيق إلى إدارة وعرض الجهات الطبية المتعاقد معها بطريقة ذكية، سريعة، وتفاعلية، مع دعم كامل للعمل بدون اتصال بالإنترنت (Offline Mode).

---

## ✨ الميزات الرئيسية (Key Features)

### 👨‍💻 واجهة المستخدم (User Experience)
* **بحث ذكي ومتقدم:** بحث نصي فوري (Debounced Search) في كافة الحقول (الاسم، التخصص، العنوان، الخ).
* **🎙️ البحث الصوتي:** دمج واجهة `SpeechRecognition` للبحث باستخدام الصوت (يدعم اللهجة المصرية `ar-EG`).
* **فلاتر ديناميكية:** تصفية النتائج بناءً على (المحافظات) و (التخصصات) مع قوائم منسدلة تدعم البحث الداخلي.
* **إجراءات سريعة (Quick Actions):** 
  * 📞 اتصال هاتفي مباشر.
  * 💬 تحويل ذكي لأرقام الهواتف (المصرية) لفتح محادثة **WhatsApp** مباشرة.
  * 🗺️ توليد رابط ديناميكي لفتح موقع الجهة الطبية على **Google Maps**.
* **دعم PWA:** إمكانية تثبيت التطبيق على الهواتف الذكية والحواسيب كبرنامج مستقل، مع دعم التخزين المؤقت (Service Workers) للعمل أوفلاين.

### 🛡️ وضع الإدارة (Admin Mode)
* **استيراد البيانات المزدوج:**
  * ☁️ **Google Sheets API:** جلب البيانات برابط مباشر من جوجل شيت مع إمكانية تحديد "أوراق عمل/محافظات" معينة لاستيرادها.
  * 📁 **ملفات Excel المحلية:** رفع وقراءة ملفات `.xlsx` مباشرة من جهاز المستخدم باستخدام `SheetJS`.
* **إدارة البيانات (CRUD):** تعديل بيانات العيادات، تعطيل/تفعيل الجهات، وحذفها محلياً.
* **📊 لوحة الإحصائيات (Analytics Dashboard):** رسوم بيانية تفاعلية باستخدام `Chart.js` لعرض توزيع الجهات حسب المحافظات وأكثر التخصصات شيوعاً.
* **تصدير البيانات (Export):**
  * 📄 **تصدير PDF:** إنشاء ملف PDF احترافي ومنسق يحتوي على ترويسة، علامة مائية، وتاريخ الإصدار باستخدام `html2pdf.js`.
  * 📊 **تصدير Excel:** تحميل البيانات المفلترة كملف إكسيل.
* **إعدادات مخصصة:** تغيير اسم التطبيق، الوصف، الشعار (Logo)، ورابط المزامنة، وحفظها في `LocalStorage`.

### ⚡ الأداء والتقنيات (Performance & Tech)
* **مزامنة لحظية (Real-time Sync):** دمج مكتبة `Pusher` للاستماع لأي تحديثات على قاعدة البيانات وتحديث واجهة المستخدم بصمت (Silent Sync).
* **تتبع ذكي (Tracking):** دمج `Google Analytics` لتتبع تفاعلات المستخدمين (مثل النقر على أزرار الاتصال أو الخريطة).
* **ترقيم الصفحات (Pagination):** نظام ترقيم ذكي للتعامل مع آلاف السجلات بدون التأثير على أداء المتصفح.

## 🛠️ التقنيات المستخدمة (Tech Stack)

* **إطار العمل:** Vue.js 3 (Composition API) عبر CDN.
* **التصميم:** Tailwind CSS + Custom CSS.
* **الخطوط والأيقونات:** Google Fonts (Cairo) + FontAwesome 6.
* **معالجة البيانات:** SheetJS (`xlsx`) لقراءة وتصدير الإكسيل.
* **تصدير PDF:** `html2pdf.js`.
* **الرسوم البيانية:** Chart.js.
* **التحديث اللحظي:** Pusher.js.
* **الاستضافة والـ CI/CD:** GitHub Pages + GitHub Actions.

## 🚀 التثبيت والتشغيل المحلي (Local Setup)

بما أن التطبيق يعتمد بالكامل على تقنيات الواجهة الأمامية (Front-end) ولا يتطلب خادم خلفي (Backend Server) معقد، يمكنك تشغيله بسهولة:

1. قم باستنساخ المستودع (Clone the repository):
   ```bash
   git clone https://github.com/yourusername/MDE.git
   ```
2. افتح المجلد:
   ```bash
   cd MDE
   ```
3. قم بتشغيل خادم محلي بسيط (لتفعيل ميزات PWA و Service Worker). يمكنك استخدام `Live Server` في VS Code، أو عبر Python:
   ```bash
   python -m http.server 8000
   ```
4. افتح المتصفح على الرابط: `http://localhost:8000`

## ⚙️ إعدادات النشر التلقائي (CI/CD & Secrets)

التطبيق مجهز بملف `deploy.yml` للنشر التلقائي على **GitHub Pages**. 
لضمان أمان مفاتيح الـ API، يقوم GitHub Actions بحقن المفاتيح أثناء عملية البناء. 

يجب عليك إضافة المتغيرات التالية في إعدادات مستودعك على جيت هاب (**Settings > Secrets and variables > Actions**):

| اسم المتغير (Secret Name) | الوصف |
|---------------------------|-------|
| `PUSHER_KEY` | مفتاح تطبيق Pusher الخاص بك للتحديث اللحظي. |
| `PUSHER_CLUSTER` | الكلاستر الخاص بتطبيق Pusher (مثل: `eu` أو `mt1`). |
| `GA_MEASUREMENT_ID` | معرف القياس الخاص بـ Google Analytics (يبدأ بـ `G-`). |

*ملاحظة: يقوم سير العمل (Workflow) أيضاً بتحديث إصدار الـ Service Worker (`sw.js`) تلقائياً في كل عملية دفع (Push) لضمان مسح الكاش القديم لدى المستخدمين.*

## 📖 دليل الاستخدام السريع

1. **إضافة البيانات لأول مرة:**
   * عند فتح التطبيق لأول مرة، ستظهر شاشة "إضافة قاعدة البيانات".
   * يمكنك إما رفع ملف Excel من جهازك، أو لصق رابط `Google Apps Script Web App` لجلب البيانات.
   * ستظهر نافذة لاختيار "الشيتات/المحافظات" المراد استيرادها.
2. **تحديث البيانات:**
   * يتم حفظ البيانات في `LocalStorage`.
   * يمكن تحديث البيانات يدوياً بالضغط على زر "تحديث" في الأعلى، أو ستتحدث تلقائياً إذا تم إرسال حدث عبر `Pusher`.
3. **وضع الإدارة:**
   * اضغط على زر "الإدارة" لإظهار أزرار التعديل، الحذف، الإحصائيات، والإعدادات.
   * من "الإعدادات" يمكنك تغيير لوجو التطبيق واسمه ليتناسب مع هويتك المؤسسية.
